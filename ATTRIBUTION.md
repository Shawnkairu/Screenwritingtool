# AI Tool Attribution

*Required for CS 372 Final Project submission. Substantive account of how AI development tools were used.*

---

## Tools Used

**Claude Code (Anthropic)** — primary development assistant throughout the project.

---

## What Was Generated

- Initial `prototype.jsx` structure was written by the developer; Claude Code assisted with migrating it into the Vite/TypeScript project structure (`src/App.tsx`, component files, `api.ts`, `types.ts`)
- Boilerplate for component files (`CharacterWeb.tsx`, `SceneCard.tsx`, `StructurePanel.tsx`, `TensionTooltip.tsx`) — translated from JSX to TypeScript with proper type annotations
- `repairJSON.ts` — logic was designed by the developer based on observed failure modes; Claude Code helped implement the bracket-tracking algorithm
- This documentation structure (README, architecture.md, evaluation.md, scoring-strategy.md, ATTRIBUTION.md)
- Backend infrastructure: `backend/main.py` FastAPI proxy with Anthropic + Ollama support, `backend/rag/retrieve.py` reranking pipeline, `backend/requirements.txt`
- Data scripts: `scripts/convert_pdfs.py` (pdfplumber conversion), `scripts/parse_screenplay.py` (scene chunk extraction), `scripts/build_rag_index.py` (embedding comparison + ChromaDB indexing)
- Evaluation scripts: `scripts/run_ablation.py` (2×2 factorial ablation), `scripts/run_prompt_comparison.py` (v1/v2/v3 prompt comparison)
- Notebooks: `notebooks/error_analysis.ipynb` (7 sections, 5 figures), `notebooks/ablation_study.ipynb`
- Writing environment component: `src/components/WritingEditorColumn.tsx` — scene/act check-in logic, formatting assistance

## What Was Designed by the Developer

- The two-layer analyst/instructor architecture — this was a deliberate product design decision, not a generated suggestion
- The instructor system prompt (`src/system-prompts/instructor.ts`) — the Socratic method, five feedback dimensions, triage intelligence, draft stage calibration, and voice guidelines were all written and iterated by the developer. This is the core IP of the product.
- The analyst JSON schema — designed to capture exactly the structural elements needed to ground the instructor's feedback
- The product vision and constraints (reactive not generative, never generates story, calibrated to skill/draft stage)
- The RAG architecture design and fine-tuning strategy

## What Needed Debugging or Substantial Rework

- **`import type` fix** — Claude Code generated `import { Analysis }` (value import) for TypeScript interfaces, which caused a Vite runtime error ("module does not provide export named 'Analysis'"). Required diagnosing the ESM/TypeScript type-stripping behavior and fixing all four component files to use `import type`.
- **CSS conflict** — The migrated app rendered a black screen because `src/index.css` (Vite boilerplate) constrained `#root` to `width: 1126px`. Required identifying that the CSS was fighting the inline `height: 100vh` layout and rewriting `index.css` to minimal resets.
- **HMR vs full reload** — After fixing the above issues, the preview still showed a blank page due to HMR state being stuck from earlier errors. Required a forced page reload to pick up the corrected module graph.
- **API key handling** — `api.ts` was generated with an empty string constant; required updating to `import.meta.env.VITE_ANTHROPIC_API_KEY` for proper Vite environment variable handling.
- **Scene heading regex** — Initial parser matched `INT./EXT.` only; iterated three times to add optional scene number prefixes, hyphen separators (Django Unchained), and em-dash separators (Inglourious Basterds). Final regex: 4 supported formats, 3,648 chunks from 35 screenplays.
- **BGE-large OOM** — Original embedding comparison used `nomic-ai/nomic-embed-text-v1` which caused 18GB buffer OOM on CPU. Required switching comparison model to `all-MiniLM-L6-v2`.
- **repairJSON empty sequence** — The error analysis notebook's truncation test crashed when no truncated strings needed repair (all parsed directly), because `min()` was called on an empty list. Fixed with a safe list comprehension check.
- **Ollama timeout** — Analyst JSON generation via `qwen2.5:7b-instruct` exceeded the hardcoded 180s backend timeout. Required increasing to 600s and implementing disk-based analyst JSON caching in the ablation script.

## How Claude Code Was Used vs. How Final Code Was Determined

Claude Code was used as a pair programmer: it could propose code quickly, but the developer reviewed, tested, and often corrected it. The product decisions — what The Instructor is, how it behaves, what the instructor voice sounds like — were entirely the developer's. Claude Code handled translation of those decisions into working TypeScript.

The instructor system prompt in particular was not generated by Claude Code. It was written by the developer and represents the primary differentiation of this product.
